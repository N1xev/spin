# Requirements: spin

**Defined:** 2026-06-02
**Core Value:** Generate a perfect, runnable Go project using charmbracelet v2 libraries with a single command.

## v1 Requirements

### Scaffolding (SCAF)

- [ ] **SCAF-01**: User can scaffold a new project with `spin new <name>` producing `./<name>/`
- [ ] **SCAF-02**: spin rejects names that are not valid Go module path segments
- [ ] **SCAF-03**: Generated project contains `go.mod`, `main.go`, `.gitignore`, `README.md`, `LICENSE`
- [ ] **SCAF-04**: Generated project has `git init` run with initial commit
- [ ] **SCAF-05**: Generated project builds with `CGO_ENABLED=0 go build ./...`
- [ ] **SCAF-06**: spin runs `go build ./...` post-scaffold and reports failures clearly
- [ ] **SCAF-07**: `spin --help` and all subcommand help render with fang styling
- [ ] **SCAF-08**: spin detects existing directory and refuses to overwrite without `--force`

### Flags (FLAG)

- [ ] **FLAG-01**: User can pass `--tui` to select the TUI project variant
- [ ] **FLAG-02**: User can pass `--cli` to select the CLI project variant
- [ ] **FLAG-03**: User can pass `--all` to union TUI + CLI stacks
- [ ] **FLAG-04**: User can pass `--bubbletea` to add `charm.land/bubbletea/v2`
- [ ] **FLAG-05**: User can pass `--bubbles` to add `charm.land/bubbles/v2`
- [ ] **FLAG-06**: User can pass `--lipgloss` to add `charm.land/lipgloss/v2`
- [ ] **FLAG-07**: User can pass `--huh` to add `charm.land/huh/v2`
- [ ] **FLAG-08**: User can pass `--glamour` to add `charm.land/glamour/v2`
- [ ] **FLAG-09**: User can pass `--glow` to add the `glow` binary tool
- [ ] **FLAG-10**: User can pass `--wish` to add `charm.land/wish/v2`
- [ ] **FLAG-11**: User can pass `--log` to add `charm.land/log/v2`
- [ ] **FLAG-12**: User can pass `--harmonica` to add `charm.land/harmonica/v2`
- [ ] **FLAG-13**: `--cobra` is default-on for CLI projects
- [ ] **FLAG-14**: `--fang` is default-on for CLI projects
- [ ] **FLAG-15**: User can pass `--viper` to opt in to `github.com/spf13/viper` config
- [ ] **FLAG-16**: User can pass `--module <path>` to override default module path
- [ ] **FLAG-17**: User can pass `--license <type>` to select MIT/Apache-2.0/none
- [ ] **FLAG-18**: Unknown flags produce a clear error with suggestion

### Templates (TMPL)

- [ ] **TMPL-01**: User can select `tui-bubbletea` template via `--template tui-bubbletea`
- [ ] **TMPL-02**: User can select `cli-cobra-fang` template via `--template cli-cobra-fang`
- [ ] **TMPL-03**: User can pass `--template-repo <url>` to clone an external template repo (depth-1, tempdir, `GIT_TERMINAL_PROMPT=0`)
- [ ] **TMPL-04**: Templates are embedded via `go:embed` and work offline by default
- [ ] **TMPL-05**: Template engine composes `_base/` + `variant_<type>/` + `lib/<lib>/` overlays (last-write-wins)
- [ ] **TMPL-06**: Generated `main.go` is a working hello-world example that runs without edits
- [ ] **TMPL-07**: Template variables include `Name`, `Module`, `Year`, `License`, `Libs []string`, `Type string`

### AI / AGENTS.md (AI)

- [ ] **AI-01**: User can pass `--ai` (alias `--agents`) to opt in to `AGENTS.md` generation
- [ ] **AI-02**: Generated `AGENTS.md` includes a `<!-- AUTOGENERATED by spin X.Y.Z -->` marker
- [ ] **AI-03**: `AGENTS.md` lists the project's charm libraries and how to extend the project
- [ ] **AI-04**: `--ai` is opt-in (not default) so users who don't want it aren't forced

### Wrappers (WRAP)

- [ ] **WRAP-01**: `spin run` runs the project (uses `air` if `.air.toml` present, else `go run`)
- [ ] **WRAP-02**: `spin build` produces a binary in `bin/<name>`
- [ ] **WRAP-03**: `spin test` invokes `prism` (with `go test` fallback when Go < 1.24 or prism missing)
- [ ] **WRAP-04**: `spin vet` runs `go vet ./...` on the whole module
- [ ] **WRAP-05**: `spin fmt` runs `gofumpt` → `goimports` → `gofmt` chain (fails loud when gofumpt missing, with `--no-strict` opt-out)
- [ ] **WRAP-06**: Each wrapper detects the preferred tool on `$PATH` and falls back to stock Go with a one-line install hint
- [ ] **WRAP-07**: Generated `.air.toml` uses `build.entrypoint` (not deprecated `build.bin`) and sets `include_ext` / `exclude_dir`
- [ ] **WRAP-08**: Generated `Taskfile.yml` has a `setup` target that installs gofumpt + goimports + prism + air

### Interactive Prompts (INT)

- [ ] **INT-01**: When flags are missing and stdin is a TTY, spin prompts via `gum` for project type / libs / template / AI
- [ ] **INT-02**: User can pass `--no-interactive` (alias `--yes`, `--batch`) to disable prompts
- [ ] **INT-03**: spin guards every TUI call with `isatty.IsTerminal(os.Stdin)` to prevent hangs in CI
- [ ] **INT-04**: When `gum` is not on `$PATH`, spin falls back to `huh v2` in-process prompts
- [ ] **INT-05**: Prompt answers are merged into the same `Project` struct as flag values (single contract)

### Tooling (TOOL)

- [ ] **TOOL-01**: Generated `go.mod` sets `go 1.25.0` when `--bubbles` is used (bubbles v2 floor)
- [ ] **TOOL-02**: Generated `go.mod` sets `go 1.23` when no bubbles
- [ ] **TOOL-03**: All generated imports use `charm.land/<lib>/v2` paths — no `github.com/charmbracelet/...` v1 paths
- [ ] **TOOL-04**: Generated project ships `.air.toml` preconfigured for the project type
- [ ] **TOOL-05**: `spin test` integration test: `spin new foo --tui --bubbletea --bubbles --lipgloss && cd foo && go build ./... && go test ./...` passes in CI

### Post-Scaffold Health (HLTH)

- [ ] **HLTH-01**: User can run `spin doctor` on a generated project to check Go version, tool presence, charm v2 path correctness, CGO=0 buildability
- [ ] **HLTH-02**: User can run `spin add <lib>` on a generated project to inject a new charm library (updates `main.go`, `go.mod`, `Taskfile.yml`)
- [ ] **HLTH-03**: User can run `spin update` to re-apply non-conflicting template changes to an existing project
- [ ] **HLTH-04**: Every generated file has a `// generated by spin X.Y.Z` header for traceability

## v2 Requirements

Deferred — not in current roadmap.

### Scaffolder UI

- **UI-01**: Scaffolder exposes an in-process TUI mode (curses-style wizard) for users who want a guided experience
- **UI-02**: `spin new` shows a live preview of the project tree before writing

### Project Distribution

- **DIST-01**: `spin release` wraps GoReleaser for the generated project
- **DIST-02**: Generated project ships Docker / OCI images
- **DIST-03**: `spin` itself distributed via Homebrew, scoop, apt

### Templates

- **TMPL-08**: `--template ssh-bubbletea` for SSH servers
- **TMPL-09**: `--template crush-agent` for crush AI agents
- **TMPL-10**: `--template bare` for no-lib starter
- **TMPL-11**: Hosted template registry with `spin template search` / `spin template install <name>`

### Flags

- **FLAG-19**: `--ssh` umbrella flag for wish projects
- **FLAG-20**: Free-form `--lib <module-path>` for arbitrary Go module adds

### Workspace

- **WS-01**: `spin workspace` manages a multi-project Go workspace (`go.work`)
- **WS-02**: Per-project config in `~/.spin.yaml`

## Out of Scope

| Feature | Reason |
|---------|--------|
| Non-charmbracelet UI frameworks (tview, ratatui, urfave/cli) | Project is opinionated about charm per user request |
| Non-Go languages (Rust, TypeScript, Python) | Wrong tool; one language per scaffolder |
| Online template marketplace | Local + `--template-repo` override is enough for v1 |
| Plugin system for custom scaffolders | Premature abstraction; defer to v2 |
| Auto-updating generated projects (live `spin update` of all files) | Out of scope per user — regenerate rather than merge |
| CI/CD pipeline generation | Project authors configure their own CI |
| Dockerfile / compose generation | Out of scope for v1 |
| Remote execution / cloud scaffolds | Local-only |
| GUI/TUI mode for the scaffolder itself | The scaffolder is a CLI; TUI is for the generated project |
| Embedding the `gum` binary | Cross-compile complications; on-`$PATH` is the install contract |
| Importing `crush` Go client (`github.com/charmbracelet/crush/internal/client`) | Internal package; treat crush as binary/config only |
| Charmbracelet v1 import paths | Deprecated; v2 only — enforced by CI grep |
