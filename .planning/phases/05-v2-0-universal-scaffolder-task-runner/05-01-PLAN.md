<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user Ôćĺ spin CLI | Untrusted flag values (project name, edition, author) |
| template render Ôćĺ filesystem | The Rust Render() output is written to disk via PostScaffold |
| git CLI Ôćĺ filesystem | `git init` and `git commit` shell out; safe because args are constants |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | rust Render() output paths | mitigate | Path-traversal guard: writeFiles rejects any path whose cleaned form escapes `dest`. Same pattern as `template.writeFiles`. Tested in `post_test.go::TestRustPost_NoTraversal` (the file created in Task 2 ÔÇö see `<files>` list). |
| T-05-02 | Information Disclosure | Cargo.toml `authors` field | accept | `author` is a user-supplied string; not a secret. Defaults to empty when unset. |
| T-05-03 | Repudiation | git commit in PostScaffold | accept | The commit message is a fixed string; no user PII. |
| T-05-04 | Denial of Service | `git init` hangs | mitigate | `git init` is non-interactive; if git is not on `$PATH`, skip with a warning rather than failing. |
| T-05-05 | Elevation of Privilege | `cargo build` invoked in Tasks() | accept | Tasks are strings the user can read via `spin run --explain <task>`. The shell runs them as the user. No sudo. |
| T-05-SC | Tampering | go module downloads | mitigate | No new dependencies introduced. |
</threat_model>
