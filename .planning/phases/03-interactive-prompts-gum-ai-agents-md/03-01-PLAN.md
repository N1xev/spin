# Plan 01 — TUI guard + non-interactive flag + prompt package skeleton

## Phase Goal (as user story)

**As a** `spin` user invoking the scaffolder from a terminal or CI, **I want to** control whether `spin new` prompts me for missing inputs, **so that** the scaffolder never hangs in scripts or CI and behaves predictably across environments.

## One-line Summary

Add the `internal/prompt` package skeleton with `ShouldPrompt`, `Fill`, and the `Canceled` error type; wire `--no-interactive`/`--yes`/`--batch` into `cmd/new.go`; relax `cobra.ExactArgs(1)` to `cobra.MaximumNArgs(1)`; map `*prompt.Canceled` to exit code 130 in `main.go`; add `go-isatty` to `go.mod`.

<objective>
Establish the TTY/CI guard chokepoint and the flag/Args plumbing required by every downstream plan. This plan ships the contract surface (`ShouldPrompt`, `Fill`, `Canceled`) but no actual prompt UI — the next plan wires huh v2 and the plan after that wires gum.
</objective>
