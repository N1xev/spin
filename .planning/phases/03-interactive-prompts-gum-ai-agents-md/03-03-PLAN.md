# Plan 03 -- gum shell-out frontend

## Phase Goal (as user story)

**As a** `spin` user with the `gum` binary on `$PATH` who runs `spin new` interactively, **I want** `spin` to use the polished gum prompts by default, **so that** I get the canonical charmbracelet look-and-feel without `spin` having to reimplement widgets.

## One-line Summary

Add the gum subprocess backend (`fillWithGum`, `gumChoose`, `gumInput`, `gumConfirm`, `gumMultiSelect`) that mirrors the huh v2 prompt sequence; wire `resolveBackend()` to pick gum when available and fall back to huh otherwise.

<objective>
Implement the primary prompt backend by shelling out to the `gum` binary for every widget. Same observable behavior as the huh backend (same questions, same defaults, same write-back to `*Project`). When gum is missing, the existing huh backend from Plan 02 takes over transparently. The backend choice is locked at `Fill` entry and never switches mid-flow.
</objective>

1. `gumRunCapture(ctx context.Context, args ...string) (string, error)` -- the only place we call `os/exec` for gum. Body:
   - `cmd := exec.CommandContext(ctx, "gum", args...)`
   - `cmd.Stdin = os.Stdin`
   - `cmd.Stdout = &stdout` (bytes.Buffer); `cmd.Stderr = &stderr` (bytes.Buffer).
   - `cmd.Cancel = func() error { if cmd.Process != nil { return cmd.Process.Kill() }; return nil }`
   - `cmd.WaitDelay = 100 * time.Millisecond`
   - On error: if `ctx.Err() != nil`, return `&Canceled{Reason: "gum canceled"}`; else return `fmt.Errorf("gum %s: %w: %s", args[0], err, stderr.String())`.
   - On success: return `strings.TrimRight(stdout.String(), "\n")`, nil.
   The 5-minute context timeout is provided by the caller (use a `context.WithTimeout(context.Background(), 5*time.Minute)` in `fillWithGum`); the function itself just respects the context.

2. `gumChoose(header string, options []string, defaultIdx int) (string, error)` -- args: `[]string{"choose", "--header", header, "--selected", strconv.Itoa(defaultIdx+1)}` + `options...`. Calls `gumRunCapture`; returns the chosen option as a string.

3. `gumMultiSelect(header string, options []string, preSelected []string) ([]string, error)` -- args: `[]string{"choose", "--no-limit", "--header", header}` + `options...`. On success, splits stdout by `"\n"`. Empty stdout returns nil (gum may print nothing if the user confirms with no selection).

4. `gumInput(header, placeholder, defaultValue string) (string, error)` -- args: `[]string{"input", "--header", header, "--placeholder", placeholder}`; if `defaultValue != ""`, append `"--value", defaultValue`.

5. `gumConfirm(prompt string, defaultYes bool) (bool, error)` -- args: `[]string{"confirm", "--default=" + strconv.FormatBool(defaultYes), prompt}`. Returns `out == "Yes"`.

6. `fillWithGum(p *scaffold.Project) error` -- mirror of `fillWithHuh` from Plan 02. Each step:
   - `askType` â†’ `gumChoose("What kind of project?", []string{"TUI -- terminal app with bubbletea", "CLI -- command-line tool with cobra + fang", "TUI + CLI -- single binary with both halves"}, 0)`; map the chosen label back to the variant name (`"tui"`, `"cli"`, `"all"`) by exact match; write `p.Type`. Skip if `p.Type != ""`.
   - `askName` â†’ `gumInput("Project name", "myapp", "")`; validate with `scaffold.IsValidGoModuleSegment`; re-prompt once on invalid; second failure returns descriptive error. Skip if `p.Name != ""`.
   - `askModule` â†’ `gumInput("Module path", "github.com/<your-org>/<name>", p.Name)`; skip if `p.Module != "" && p.Module != p.Name`.
   - `askLibs` â†’ `gumMultiSelect("Pick libraries", libDisplays, preSelected)`; convert back to `[]string` of Names (build reverse map once at package init from `LibCatalog`); write back to `p.Libs` (sorted+deduped) AND the corresponding bools.
   - `askLicense` â†’ `gumChoose("License", []string{"MIT", "Apache-2.0", "None"}, licenseIdx)`; map back to lowercase key. Skip if `p.License != "" && p.License != "mit"`.
   - `askTemplate` â†’ `gumChoose("Template", templateOptionsForType(p.Type), 0)`; skip if `p.Template != "" && p.Template != "tui-bubbletea"`.
   - `askTemplateRepo` â†’ `gumInput("External template repo URL", "(skip to use embedded templates)", "")`; validate; empty means skip.
   - `askAI` â†’ `gumConfirm("Generate AGENTS.md for AI assistants? Adds an AGENTS.md describing the project's libraries and how to extend them.", true)`; `out == "Yes"` sets `p.AI = true`.

Each `gum*` call goes through the package-level `gumRunner` variable, NOT directly through `gumRunCapture`. This is the seam tests use to mock. Default `gumRunner = gumRunCapture`. The test file mutates it in TestMain / per-test.

Import `context`, `os/exec`, `strings`, `strconv`, `time`, `github.com/example/spin/internal/scaffold`.
