<objective>
Implement the core of `spin update` (per CONTEXT D-06..D-08, D-10): a universal Go dep updater. Three sub-tasks, three files:
1. Parse a `go.mod` to extract direct (or all, via `--all`) dependencies with their current pinned version.
2. Resolve the next available stable and latest versions for each module by hitting the public Go module proxy (`https://proxy.golang.org/<module>/@v/list`).
3. Apply selected versions with `go get module@version` per dep, then a single `go mod tidy`, then a `CGO_ENABLED=0 go build ./...` smoke test. No `go test ./...` (per D-10).

The huh v2 form, the cobra subcommand, and the `--all` flag are handled in Plan 04-04. This plan is the engine; Plan 04-04 is the UI.

Purpose: HLTH-03, rewritten as a dep updater (the original "template merge" reading is retired per D-12).

Output: `internal/update/` package with 3 source files + 3 test files, parse/resolve/apply fully tested against fixtures.

Create `internal/update/resolve.go`:
- Package `update`.
- Define an interface `ModuleProxy` with one method: `ListVersions(ctx context.Context, module string) ([]string, error)`. The default impl is `HTTPMirror` which fetches `https://proxy.golang.org/<module>/@v/list` (the module path is URL-escaped via `url.PathEscape`), reads the body, splits on `\n`, and trims whitespace. A 10s timeout is enforced via `http.Client{Timeout: 10*time.Second}`. Set `User-Agent: spin/<version>` (read from `internal/version.Version`). This indirection lets tests inject a fake proxy.
- Define a struct `Resolver` holding a `ModuleProxy` and a `Logger` (or `io.Writer` for simple cases).
- Define `Resolver.Resolve(ctx context.Context, deps []Dep) ([]Dep, error)`:
  1. For each input `Dep`, call `proxy.ListVersions(ctx, dep.Module)`. If the proxy returns 404 (the module is local-only, e.g. an unpublished module or a replace directive target), set `NewStable = dep.Old` and `NewLatest = dep.Old` and continue (no error).
  2. Filter the version list:
     - `newStable` candidates: versions where `semver.Prerelease(v) == ""` (i.e. no `-beta`, `-rc`, `-alpha`, `-pre` suffix). Pick the highest by `semver.Compare`.
     - `newLatest` candidates: all semver-valid versions, pick the highest by `semver.Compare`.
  3. If a category's candidate set is empty (e.g. only pre-releases exist for a fresh module), set that field to `dep.Old` (no upgrade available).
  4. Return the enriched `[]Dep` with `NewStable` and `NewLatest` populated.
- Use `semver.IsValid` and `semver.Compare` from `golang.org/x/mod/semver` (already a transitive dep). Versions from the proxy list come as `v1.2.3` (with the `v` prefix); `semver.Compare` requires the canonical form (`v1.2.3` is already canonical for tagged releases, but pre-releases like `v1.2.3-beta.1` need to be lowercased â€” `semver.Canonical` handles that).
- For module paths that need URL escaping: `url.PathEscape("charm.land/bubbletea/v2")` returns `"charm.land/bubbletea/v2"` (no change because slashes are not escaped by `PathEscape`; but uppercase letters would be, and Go module paths are case-sensitive lowercase). `PathEscape` is safer than `QueryEscape` because it leaves `/` as-is, which the proxy expects.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User shell â†’ `spin update` | Untrusted `args` and the cwd's `go.mod` (could be hand-rolled, malicious, or huge). |
| `spin update` â†’ `proxy.golang.org` | The CLI makes outbound HTTPS requests to fetch version lists. The proxy is operated by the Go team and signed via the Go checksum database (sum.golang.org), but our v1 client does NOT verify the proxy's signed tree â€” it trusts the URL. |
| `spin update` â†’ local `go` toolchain | The CLI shells out to `go get`, `go mod tidy`, `go build` with module paths derived from go.mod. Trust: these are first-party tools, but the argv comes from the user's go.mod. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-14 | Tampering | `go.mod` parse | mitigate | Use `modfile.Parse` (the official parser) â€” does not eval, does not shell. |
| T-04-15 | Tampering | `go get module@version` argv | mitigate | Static argv construction (`exec.Command("go", "get", module, version)`), no shell, no interpolation of user input. The `module` value comes from a previously-parsed go.mod line, not from the command line. |
| T-04-16 | Tampering | Network fetch to proxy.golang.org | mitigate | HTTPS only (no fallback to HTTP). 10s timeout via `http.Client.Timeout`. The version list response is a plaintext newline-separated list â€” the worst a MITM can do is feed us a list with versions that don't exist, but `go get module@nonexistent` will then fail with a non-zero exit, which our Apply surfaces as an error. |
| T-04-17 | Tampering (Go checksum DB bypass) | We do not call `sum.golang.org` directly; `go get` does, internally. | accept | The Go toolchain handles sumdb verification for module downloads. We rely on `go get`'s own integrity checks rather than reimplementing them. |
| T-04-18 | Information Disclosure | The HTTP request to proxy.golang.org reveals the modules we depend on | accept | The Go team operates the proxy knowing its access patterns are public. We do not add any extra headers that leak user identity. User-Agent is `spin/<version>` (the project's own identifier, no PII). |
| T-04-19 | Elevation of Privilege | A malicious go.mod could trick `go get` into running arbitrary code via a tool dependency | accept | Go's `go get` of a `// indirect` tool dep installs binaries to `$GOPATH/bin`. This is standard Go behavior, not a new attack surface introduced by spin. We document this in the Apply function's godoc. |
| T-04-20 | Denial of Service | Slow network / huge go.mod / many deps | mitigate | 10s HTTP timeout per module fetch. The exec calls (go get, tidy, build) are bounded only by the user's patience â€” for v1, no enforced timeout. |
| T-04-21 | Repudiation | Apply must surface build errors to the user | mitigate | The error from `go build` is wrapped with the combined stdout+stderr, capped at the natural command output (no truncation needed; `go build` output is small). |
| T-04-SC | Tampering | No new direct dependencies | n/a | `golang.org/x/mod/semver` is already a transitive dep (via modfile, which is a transitive dep of x/mod). The net/http stdlib is used. No `go get` adds a new direct dep in *spin*'s own go.mod. |
</threat_model>
