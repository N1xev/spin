Create `internal/update/form.go`:
- Package `update`.
- Define a constant string for the "Skip" sentinel. The form's Select stores a string key per option; "skip" is the canonical "do nothing" choice. Other keys: "stable" and "latest".
- Define a struct `UpdateChoice` capturing one form result: `Dep Dep` (with the original `Old` and the resolved `NewStable`/`NewLatest`) and `Action string` (one of "skip", "stable", "latest"). The form writes to a slice of these via a `map[string]*string` of pointers keyed by module path.
- Define `PromptForUpdate(ctx context.Context, opts PromptOptions) error`:
  - `PromptOptions` struct: `GoModPath string` (path to go.mod; required), `IncludeIndirect bool` (default false), `Log io.Writer` (where to write progress), `Proxy ModuleProxy` (defaults to `HTTPMirror{}` if nil), `In io.Reader` (default `os.Stdin`), `Out io.Writer` (default `os.Stdout`).
  1. Call `ListDeps(opts.GoModPath, opts.IncludeIndirect)`. If the result is empty, write "no deps to update\n" to `opts.Log` and return nil.
  2. Call `Resolver{Proxy: opts.Proxy}.Resolve(ctx, deps)`. Propagate the error.
  3. Check `isatty.IsTerminal(os.Stdin)` (the file descriptor, not the `opts.In` reader; the prompt always reads from the controlling terminal). If not a TTY, call `printNonTTYTable(deps, opts.Out)` which renders a 3-column table (`module | old | stable | latest`) and returns `fmt.Errorf("update: stdin is not a TTY; re-run interactively or pass --batch (not yet implemented)")`. This prevents hangs in CI per INT-03.
  4. Build the huh form. For each dep, allocate `var choice string` (the action key) and add a `huh.NewGroup(huh.NewSelect[string]().Title(fmt.Sprintf("%s  (%s → %s / %s)", dep.Module, dep.Old, dep.NewStable, dep.NewLatest)).Options(huh.NewOption("Skip", "skip"), huh.NewOption("newStable: "+dep.NewStable, "stable"), huh.NewOption("newLatest: "+dep.NewLatest, "latest")).Value(&choice))`. Default the `choice` to "stable" *before* calling `huh.NewSelect` -- assign `choice = "stable"` directly so the Select renders with "newStable" pre-selected. (Phase 3 noted gum doesn't support pre-selection, but huh's `.Value(&v)` does pre-populate the displayed value when `v` is set.)
  5. Call `form.Run()`. If `errors.Is(err, huh.ErrUserAborted)`, return a `*prompt.Canceled` (or define a local `ErrCanceled` in this package -- DO NOT depend on the prompt package from update to keep the dependency graph one-way; update → prompt would be a layering inversion since prompt already depends on scaffold). Use a local `ErrCanceled = errors.New("update: user canceled")`.
  6. After the form returns, build the `[]UpdateChoice` from the form's string keys. For each dep, if the action is "stable", overwrite `dep.Old = dep.NewStable` (so Apply sees a target); if "latest", overwrite with `dep.NewLatest`; if "skip", drop from the apply list. Call `Apply(applyList, filepath.Dir(opts.GoModPath), opts.Log)`.

- Define `printNonTTYTable(deps []Dep, w io.Writer) error`: format a 4-column text table (module, old, stable, latest) using `text/tabwriter` for column alignment. Header row: `MODULE  OLD  STABLE  LATEST`. One row per dep. If `NewStable` equals `Old`, print `(current)` in the stable column. Same for latest. Truncate module names longer than 50 chars with an ellipsis.

- Use `github.com/mattn/go-isatty` for the TTY check (already a direct dep per go.mod).

Create `internal/update/form_test.go`:
- `TestPromptForUpdate_NonTTY_PrintsTable_NoHang` -- redirect `os.Stdin` to a pipe (so isatty returns false). Call `PromptForUpdate` with a temp go.mod fixture and a fake `ModuleProxy` that returns 3 versions. Assert the function returns an error (because non-TTY), and the `Out` writer contains the table header and 3 module rows. Use a small timeout (1s via `context.WithTimeout`) to prove no hang.
- `TestPromptForUpdate_EmptyDeps_ReturnsNil` -- fixture go.mod with zero direct requires. Assert `PromptForUpdate` returns nil and writes "no deps to update" to the log writer.
- `TestPromptForUpdate_BuildsFormForEachDep` -- extract the form-building logic into a helper `buildForm(deps []Dep) *huh.Form` (exported for tests) and assert it returns a non-nil form whose group count equals `len(deps)`. Don't actually run the form in the test (huh needs a TTY).
- `TestPrintNonTTYTable_FormatsCurrentAsCurrent` -- call with a dep where NewStable == Old. Assert the table contains the literal "(current)" string in the stable column.
- `TestPrintNonTTYTable_TruncatesLongModuleNames` -- call with a dep whose module name is 80 chars. Assert the row is ≤ 60 chars (module column truncated to 50 + 4 chars of padding).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User shell → `spin update` | Untrusted cwd, untrusted `go.mod` (could be hand-rolled). |
| `spin update` → huh v2 form | The form reads from `os.Stdin`. If stdin is not a TTY, the form hangs -- mitigated by the isatty guard. |
| `spin update` → `go.mod` write | Apply does not edit go.mod directly; it shells out to `go get` which does. Trust: first-party tool. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-22 | Denial of Service | huh form hangs on non-TTY stdin | mitigate | isatty.IsTerminal(os.Stdin) check before building the form. Non-TTY path renders a table and returns a non-zero exit. TestPromptForUpdate_NonTTY_PrintsTable_NoHang proves this with a 1s context timeout. |
| T-04-23 | Tampering | User could craft a huh form that runs arbitrary code | accept | huh v2 is a pure-UI library; it does not exec anything. The only side effects are (a) writing the chosen string keys back to our pointers and (b) the subsequent Apply, which we already vetted in Plan 04-03. |
| T-04-24 | Tampering | `--all` flag could include a malicious indirect dep | accept | We only ever read indirect deps from a parsed go.mod (already vetted by modfile.Parse in Plan 04-03). The list of indirect deps cannot exceed what is in the file. |
| T-04-25 | Repudiation | User cancellation must not be treated as a hard error | mitigate | `huh.ErrUserAborted` is mapped to a local `ErrCanceled` returned from PromptForUpdate. cobra + fang render it as a styled "canceled" message and the process exits non-zero but with a clear reason (not a 500-style crash). |
| T-04-26 | Information Disclosure | The form displays the full module path on screen | accept | This is the intended UX -- the user is choosing what to upgrade and must see the module paths. |
| T-04-SC | Tampering | No new direct dependencies in this plan | n/a | `charm.land/huh/v2` and `github.com/mattn/go-isatty` are already in go.mod as direct deps from Phase 3. No `go get` runs in this plan. |
</threat_model>
