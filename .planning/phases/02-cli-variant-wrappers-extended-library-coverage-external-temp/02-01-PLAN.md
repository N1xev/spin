# Plan 02-01: Foundation refactor

## Phase Goal

**As a** Go developer, **I want to** generate a working CLI or TUI project scaffolded with any charmbracelet v2 library, **so that** I can start building immediately without manual setup of dependencies, tooling, or build configs.

This plan is the foundation. It fixes the four things the v1 research got wrong (per the Phase 2 research §1 / §2 / §7) so that everything else in Phase 2 builds on solid ground:

1. **Pin freshness**: bump all 11 lib pins (10 `charm.land/<lib>/v2` + 1 `github.com/charmbracelet/harmonica` per the §2.1 correction) to the latest stable per the research.
2. **Go version floor**: collapse the v1 `{{if hasBubbles .}}1.25.0{{else}}1.23{{end}}` dead branch to `go 1.25.0` always. Per research §2.2, every charm v2 lib requires 1.25.0+ transitively; the 1.23 branch is unreachable.
3. **Path-traversal guard**: harden `emit()` so a template that renders `{{.Name}}` to `../../etc/passwd` cannot write outside the project root.
4. **Grep suite refinement**: stop the blanket `github\.com/charmbracelet/` ban from false-positiving on `github.com/charmbracelet/harmonica` and `github.com/charmbracelet/glow/v2` (which the research §2.1 confirms are still the current paths for those two libs).

## Context

Phase 1 shipped the Walking Skeleton on June 2, 2026. `internal/scaffold/versions.go` `DefaultPins` has four fields (Bubbletea v2.0.0, Lipgloss v2.0.0-beta.2, Bubbles v2.0.0, Log v2.0.0) -- the lipgloss pin is the explicit "v2.0.0-beta.2 module-path mismatch" entry in `~/.claude/memory/MEMORY.md`. The `_base/go.mod.tmpl` has a `{{if hasBubbles .}}1.25.0{{else}}1.23{{end}}` branch that is dead code (per the research §2.2 the 1.23 branch never fires because every charm v2 lib the v1 templates emit requires 1.25.0 transitively).

The grep suite in `scripts/check-v1-leaks.sh` blanket-bans `github\.com/charmbracelet/` which is wrong for harmonica and glow (still on github.com per the v1 → v2 research correction). The emit() function in `scaffold.go` writes whatever path the overlay walker returns without verifying it stays inside `./<name>/`.

This plan fixes all four before any Phase 2 work uses the engine.

### Task 3: Add path-traversal guard in `emit()`

**Files:** `internal/scaffold/scaffold.go`

**Action:** Modify `emit()` (the function that writes rendered files to `./<name>/`) to verify each rendered relative path resolves inside the project root before writing. The check uses `filepath.Clean` and `strings.HasPrefix` so a template that renders `{{.Name}}` to `../../etc` is rejected with a descriptive error.

The check is portable: `filepath.Separator` is `\` on Windows and `/` on POSIX; both sides of the comparison use the same separator.

The guard sits in the existing `for rel, content := range files` loop in `emit()`, just before the `os.WriteFile` call. The error message names the offending rendered path and the project root for fast debugging.

**Verify:** `go test ./internal/scaffold/... -run TestEmit_PathTraversal` passes (this test is added as part of Task 5). The test creates a `Project{Name: "myapp"}`, calls `emit` directly with a map containing a key like `../../etc/passwd`, and asserts the returned error mentions "path traversal".

**Commit:** `fix(02): reject template output paths that escape the project root`

### Task 4: Refine `scripts/check-v1-leaks.sh` v1-leak patterns

**Files:** `scripts/check-v1-leaks.sh`

**Action:** Replace the single `'github\.com/charmbracelet/'` regex in `GO_PATTERNS` with an explicit deny-list of the 8 modules that migrated to `charm.land/<lib>/v2` (bubbletea, lipgloss, bubbles, huh, glamour, wish, log, fang -- both the bare form and the `/v2` form). Leave a comment block explaining that `harmonica` and `glow` are intentionally NOT in the list because they have not migrated.

The deny-list pattern is `github\.com/charmbracelet/<lib>"` (closing quote anchor) so an import like `github.com/charmbracelet/harmonica` is not matched, but a bare `github.com/charmbracelet/bubbletea` import is. This requires both forms per lib (`<lib>"` and `<lib>/v2"`).

**Verify:** `bash scripts/check-v1-leaks.sh ./internal/scaffold/templates` exits 0 (clean); a synthetic test fixture with `import "github.com/charmbracelet/harmonica"` passes (false positive removed); a synthetic test fixture with `import "github.com/charmbracelet/bubbletea"` fails. These three checks become `TestGrepV1Leaks_AllowsHarmonica` and `TestGrepV1Leaks_CatchesV1Import` (the latter already exists; add the former).

**Commit:** `fix(02): refine v1-leak grep to per-module deny-list (allow harmonica + glow)`

- Do NOT blanket-allow `github.com/charmbracelet/` in the grep suite -- the per-module deny-list is the whole point of this refactor.
- Do NOT remove the `_base/go.mod.tmpl` `{{if hasBubbletea .}}` block for the bubbletea pin -- it's still the right pattern for Phase 1's three active libs.
