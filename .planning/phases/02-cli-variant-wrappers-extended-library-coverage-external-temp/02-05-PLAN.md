# Plan 02-05: Restructure scaffold templates to idiomatic Go project layout

## Phase Goal (as user story)

**As a** `spin` user who scaffolds a new project, **I want** the generated code to look like a real charmbracelet v2 example app (interleaved libs in `internal/app/*.go` for TUI, `internal/cmd/*.go` for CLI) using the correct Go project layout (`cmd/<name>/main.go` + `internal/`), **so that** I can read the scaffolded code as a real example, learn the idioms in-place, and immediately extend it -- rather than staring at a parts catalog of one-file-per-lib overlays.

## Why

Current Phase 2 templates use a "file-per-lib overlay" architecture (`lib/huh/huh.go.tmpl`, `lib/wish/wish.go.tmpl`, etc.) where each enabled lib drops its own file into the project root, and the variant `main.go` calls into those files via stub function references (`huhExample()`, `wishExample()`, `glamourExample()`). This produces a parts catalog, not an idiomatic Go app. The scaffolded code does not look like a real charmbracelet example -- the libs do not compose inside `Update()` / `View()`; they live in separate files that the main.go merely delegates to. Also, `main.go` at the project root is not the canonical Go layout.

## What changes

- New TUI layout: `cmd/<name>/main.go` (thin) + `internal/app/{app,update,view,keys}.go` + `internal/ui/styles.go`
- New CLI layout: `cmd/<name>/main.go` (thin) + `internal/cmd/{root,hello,readme,ssh}.go` + `internal/config/config.go`
- New all-variant layout: composes both
- All lib code is inlined as `{{ if has* . }}` blocks in the variant files -- no more `lib/<name>/<name>.go` overlays
- Walker substitutes `<name>` / `_name_` placeholder in output paths with `p.Name` so `cmd/_name_/main.go.tmpl` → `cmd/myapp/main.go`

## One-line Summary

Replace file-per-lib overlays with a single Go-idiomatic project tree per variant: `cmd/<name>/main.go` thin entry + `internal/app/{app,update,view,keys}.go` (TUI) or `internal/cmd/{root,hello,readme,ssh}.go` (CLI), with all charmbracelet libs inlined as `{{ if has* . }}` blocks so the generated app looks like a real example.

## Tasks

### Task 1: Add `<name>` / `_name_` placeholder substitution to template walker

**Why:** `cmd/_name_/main.go.tmpl` template files need to write to `cmd/myapp/main.go` (or whatever the project is named). The walker currently just strips `.tmpl` from the relative path; we need it to substitute the project name.

**Files:** `internal/scaffold/template.go`

**Steps:**
1. In `renderToMap()`, after computing `outKey := strings.TrimSuffix(filepath.ToSlash(rel), ".tmpl")`, add `outKey = strings.ReplaceAll(outKey, "_name_", p.Name)`.
2. Use `_name_` (not `<name>`) because `<>` are valid in filenames but harder to type in editors. Document the convention in the function comment.
3. Add a unit test in `internal/scaffold/template_test.go` (or new file) that scaffolds with a project name and asserts the output map has the key `cmd/myapp/main.go` not `cmd/_name_/main.go`.
4. Edge case: if the project name happens to be `_name_` literally, the substitution is a no-op (literally the same string). Acceptable.

**Acceptance:** `go test ./internal/scaffold/... -run TestRenderToMap_NameSubstitution` passes; existing tests still pass.

**`cmd/_name_/main.go.tmpl` (thin entry):**
```go
// Code generated by spin {{.SpinVer}}. DO NOT EDIT.

package main

import (
	"fmt"
	"os"

{{- if hasLog .}}
	"charm.land/log/v2"
{{- end}}
	"{{.Module}}/internal/app"
)

func main() {
{{- if hasLog .}}
	logger := log.New(os.Stderr)
	log.SetDefault(logger)
	log.SetLevel(log.InfoLevel)
{{- end}}
	if _, err := app.Run(); err != nil {
{{- if hasLog .}}

{{- else}}

{{- end}}
		os.Exit(1)
	}
}
```

**`internal/app/app.go.tmpl` (Model + New + Init):** Contains `type Model struct{}` with conditional fields (`{{ if hasBubbles . }}spinner spinner.Model{{ end }}`, `{{ if hasLipgloss . }}styles ui.Styles{{ end }}`, `{{ if hasHarmonica . }}springPos float64{{ end }}`, `{{ if hasHuh . }}name string{{ end }}`). `New()` returns `*Model` with field initialization. `Init() tea.Cmd` returns the initial spinner tick when --bubbles, else `nil`. `Run()` is the `func main` entry that calls `tea.NewProgram(New()).Run()`.

**`internal/app/update.go.tmpl` (Update with embedded lib logic):** This is the heart of the restructure. The Update function has:
- Always: q/ctrl+c/esc quit
- `{{ if hasBubbles . }}`: `case spinner.TickMsg: m.spinner, cmd = m.spinner.Update(msg); return m, cmd`
- `{{ if hasLipgloss . }}`: `case tea.WindowSizeMsg: m.styles = ui.NewStyles(msg.Width); return m, nil`
- `{{ if hasHuh . }}`: `case "n": form := huh.NewForm(huh.NewGroup(huh.NewInput().Title("Your name?").Value(&m.name))); _ = form.Run(); m.greeted = true; return m, nil`
- `{{ if hasGlamour . }}`: `case "g": m.helpMarkdown = renderHelp(); return m, nil` (where `renderHelp` is a local func that uses glamour)
- `{{ if hasHarmonica . }}`: `case "s": return m, m.tickSpring()` and `case harmonicaTickMsg: m.springPos = msg.pos; return m, nil`
- `{{ if hasLog . }}`: `case "l": log.Info("counter", "count", m.count); return m, nil`

The local helper functions (huh.NewForm, glamour.NewTermRenderer, harmonica.NewSpring, spinner.New) are all defined in the same file via `{{ if has* . }}` blocks. No `lib/huh/huh.go` import.

**`internal/app/view.go.tmpl` (View with embedded rendering):**
- Compose spinner view, lipgloss styled title, counter, help text, animated spring value
- `{{ if hasBubbles . }}` shows the spinner
- `{{ if hasLipgloss . }}` wraps the title in `m.styles.Title.Render(...)`
- `{{ if hasGlamour . }}` shows the rendered help markdown in a section
- `{{ if hasHarmonica . }}` shows `Spring: %.2f`
- `{{ if hasHuh . }}` shows `Greeted: <name>` when `m.greeted`

**`internal/app/keys.go.tmpl` (KeyMap, always for TUI):**
- Standard bubbletea help.Component KeyMap: `up`, `down`, `quit`, `help`, plus conditional entries (`name`, `glamour`, `spring`, `log`)

**`internal/ui/styles.go.tmpl` (lipgloss, always for TUI):**
- The same content as the current `lib/lipgloss/internal/ui/styles.go.tmpl`

**Acceptance:** Generated `cmd/myapp/main.go` imports `internal/app` and calls `app.Run()`. Generated `internal/app/update.go` contains `huh.NewForm(...)` inline when --huh is set. `go build ./...` succeeds on a TUI scaffold with all libs enabled.
