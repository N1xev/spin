## Minimal example

```toml
name = "go-cli"
version = "0.1.0"
description = "A minimal Go CLI"
language = "go"
type = "cli"

[params]
project_name = { type = "text", prompt = "Project name" }

[[post]]
run = "go mod init {{ .project_name }}"
```

The template also needs a `_base/` directory next to `spin.toml` (see [template anatomy](/templates/anatomy)).

## Top-level fields

| Field | Required | Description |
| --- | --- | --- |
| `name` | yes | Template name. Used in messages and the `--print-params` output. |
| `version` | no | Template version, e.g. `0.1.0`. |
| `description` | no | Short description shown in search results. |
| `type` | no | Project shape, e.g. `cli`, `tui`, `lib`. Free-form; used in search. |
| `language` | no | Primary language, e.g. `go`, `rust`, `ts`. Free-form; used in search. |
| `license` | no | License identifier, e.g. `MIT`. |
| `repository` | no | Source URL for the template. |
| `min_spin_version` | no | Minimum spin CLI version required, e.g. `0.2.0`. Emits a warning if the running spin is older. |
| `tags` | no | Array of strings, e.g. `["web", "api"]`. Used in search. |
| `exclude` | no | Array of glob patterns. Files whose rendered path matches are skipped. |

## `[author]`

Optional. Recognised fields:

```toml
[author]
name = "Sam"
email = "sam@example.com"
url = "https://sam.example.com"
```

## `[params]`

Each key becomes a question during `spin new`. The value is an inline table with at least a `type`:

```toml
[params]
project_name = { type = "text", prompt = "Project name" }
edition = { type = "select", options = ["2021", "2024"], default = "2021" }
features = { type = "multiselect", options = ["http", "grpc"], default = ["http"] }
port = { type = "number", default = 8080, min = 1000, max = 65535 }
private = { type = "bool", prompt = "Make the repository private?", default = false }
```

### Common fields

| Field | Description |
| --- | --- |
| `type` | Param type (see below). Defaults to `text` if omitted. |
| `prompt` | Question text. Defaults to the param name. |
| `default` | Default value. |

### Param types

| Type | Stored as | Notes |
| --- | --- | --- |
| `text` | string | Single-line input. |
| `textarea` | string | Multi-line input. |
| `number` | int | `min` and `max` are optional. |
| `select` | string | Requires `options`. `default` must be one of the options. |
| `multiselect` | `[]string` | Requires `options`. `default` is an array. |
| `bool` | bool | Rendered as a yes/no confirm. |
| `path` | string | File picker. Use a file path. |
| `secret` | string | Hidden input. No `default` is shown. |

All values are available in templates as `{{ .param_name }}`. `name` and `project_name` are always injected from the project name supplied to `spin new`.

## `[[pre]]`

Pre-scaffold shell steps run in the generated project directory after params are resolved but before files are rendered or written. Each step is a `run` string rendered against the resolved values:

```toml
[[pre]]
run = "mkdir -p {{ .project_name }}/cmd"

[[pre]]
run = "go mod init {{ .project_name }}"
```

Steps run in order. If one fails, the hook stops and `spin new` returns the error before any files are written. The rendered command is run via `sh -c` in the project directory.

Use `{{ quote .param_name }}` to safely insert a value into a shell command.

## `[[post]]`

Post-scaffold shell steps run in the generated project directory, after files are written but before the output `spin.toml` is removed. Each step is a `run` string rendered against the resolved values:

```toml
[[post]]
run = "go mod init {{ .project_name }}"

[[post]]
run = "git init && git add -A && git commit -m 'initial'"
```

Steps run in order. If one fails, the hook stops and `spin new` returns the error. The rendered command is run via `sh -c` in the project directory.

Use `{{ quote .param_name }}` to safely insert a value into a shell command.

## `exclude`

Glob patterns matching the rendered relative path of files that should be omitted from the output. The `.tmpl` extension is stripped before matching.

```toml
exclude = ["*.draft.md", "docs/internal/*"]
```

## `[[include]]`

Positive conditional rules for including files or whole directories. Each rule has a `path` glob and an optional `if` template rendered against the resolved values. When at least one `[[include]]` rule exists, only files matching a true rule are copied or rendered. Files that do not match any rule are skipped, and directories with no matching rule are pruned entirely.

```toml
[params]
ci = { type = "bool", prompt = "Include CI?", default = true }
features = { type = "multiselect", options = ["auth", "grpc"], default = [] }

[[include]]
path = ".github/**"
if = "{{ .ci }}"

[[include]]
path = "auth/**"
if = "{{ has .features \"auth\" }}"

[[include]]
path = "grpc/**"
if = "{{ has .features \"grpc\" }}"
```

A rule with an empty or omitted `if` always includes matching paths. If no `[[include]]` rules exist, all non-excluded files are included (backward compatible).

`path` supports `*` for a single segment and `**` for any number of segments, relative to `_base/`.

## Template functions

Files in `_base/` are rendered with Go's `text/template`. You get the full standard library: `{{ range }}`, `{{ if }}`, `{{ with }}`, variables, pipelines, and all built-in template functions.

### Built-in Go functions

Go's `text/template` ships with these functions out of the box:

| Function | Example | Notes |
| --- | --- | --- |
| `and` | `{{ and .foo .bar }}` | Boolean AND. |
| `or` | `{{ or .foo .bar }}` | Boolean OR. |
| `not` | `{{ not .private }}` | Boolean NOT. |
| `eq` | `{{ eq .license "MIT" }}` | Equality. |
| `ne` | `{{ ne .license "MIT" }}` | Inequality. |
| `lt` / `le` / `gt` / `ge` | `{{ gt .port 1024 }}` | Numeric comparisons. |
| `len` | `{{ len .features }}` | Length of string, array, slice, or map. |
| `index` | `{{ index .features 0 }}` | Index into array/slice/map. |
| `slice` | `{{ slice .features 1 }}` | Sub-slice. |
| `print` / `printf` / `println` | `{{ printf "%q" .name }}` | Format strings. |
| `call` | `{{ call .fn }}` | Call a function value. |

See the [Go text/template documentation](https://pkg.go.dev/text/template) for the complete list.

### spin helpers

`spin` adds these functions on top:

| Function | Example | Result |
| --- | --- | --- |
| `upper` | `{{ upper .name }}` | `MYPROJECT` |
| `lower` | `{{ lower .name }}` | `myproject` |
| `title` | `{{ title .name }}` | `Myproject` |
| `trim` | `{{ trim .name }}` | Trims whitespace. |
| `join` | `{{ join .tags ", " }}` | Joins a list with a separator. |
| `default` | `{{ default "foo" .name }}` | Returns the first value if the second is empty. |
| `snake_case` | `{{ snake_case .name }}` | `my_project` |
| `kebab` | `{{ kebab .name }}` | `my-project` |
| `quote` | `{{ quote .name }}` | Shell-safe single-quoted string. |
| `now` | `{{ now "2006" }}` | Current time in the given layout. |
| `contains` | `{{ if contains .tags "api" }}...` | Substring check. |
| `has` | `{{ has .features "auth" }}` | `true` if a `[]string` contains the item. |
| `not_has` | `{{ not_has .features "auth" }}` | `true` if a `[]string` does not contain the item. |
| `one_of` | `{{ one_of .license "MIT" "Apache-2.0" }}` | `true` if the value equals any of the items. |

## Conditional content

Use Go template control structures to include or skip content inside a file:

```tmpl
{{ if .use_cobra }}
package cmd

import "github.com/spf13/cobra"
{{ else }}
package main
{{ end }}
```

```tmpl
{{ range .features }}
// {{ . }} enabled
{{ end }}
```

To gate whole files or directories, use `[[include]]` rules.
