## Requirements

- A `spin.toml` file at the root.
- A `_base/` directory with at least one file.
- The template must render without errors for its default param values.

## Versioning

Use git tags or branches. `spin add` and `spin new` shallow-clone the default branch by default. If you want users to pin a specific version, publish separate branches or repos.

## README conventions

Include a one-line description, the install command, and the params users will be asked for:

```markdown
# go-cli-template

A minimal Go CLI starter for spin.

```sh
spin new myapp --template https://github.com/example/go-cli-template.git
```

## Params

- `project_name`: project name (default: myproject)
- `license`: MIT, Apache-2.0, BSD-3-Clause, or Proprietary (default: MIT)
```

## Testing a template locally

```sh
spin new test-output --template ./my-template --dry-run
spin new test-output --template ./my-template --param project_name=test
```

## Distribution options

- GitHub/GitLab repo: `spin new myapp --template https://github.com/user/repo.git`
- Local path: `spin new myapp --template ~/templates/my-template`
- Registry: add the repo to a spin registry so users can `spin search` and `spin add <alias>/<id>`.
